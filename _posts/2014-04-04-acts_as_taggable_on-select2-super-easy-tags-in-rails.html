I've had two projects in the past little bit where I wanted user input tags for objects. I also wanted to have typed tags - tags of different categories applied to the same object. I found that acts_as_taggable_on (Rails gem) and select2 (jQuery plugin) work really nicely together and make this an easy task.\r\n\r\nLet's assume that we have an ActiveRecord model for a Post. Follow directions to install acts_as_taggable_on (add to Gemfile, initialize and run migration). Then add two different tag types to Post like this:\r\n\r\n[sourcecode language=\"ruby\"]\r\nclass Post < ActiveRecord::Base\r\n \ acts_as_taggable # basic usage - gives us 'tags'\r\n acts_as_taggable_on :topics\r\nend\r\n[/sourcecode]\r\n\r\nWith this, the Post model is ready to store tags and topics.\r\n\r\n[sourcecode]\r\nirb> Post.first.tags << 'super fun'\r\n[/sourcecode]\r\n\r\nTo push this to the users, we want a nice UI for users to enter tags/topics. This is where Select2 comes in. In your form for adding/editing posts, add two input boxes, one for topics and one for tags\r\n\r\n[sourcecode]\r\n# using haml templating\r\n.input-row\r\n \ = f.label :tags\r\n = f.text_field :tag_list\r\n.input-row\r\n = f.label :topics\r\n \ = f.text_field :topics_list\r\n[/sourcecode]\r\n\r\nThen attach the select2 plugin to the form elements. Make sure you've got jQuery and select.js included on the page.\r\n\r\n[sourcecode language=\"javascript\"]\r\n// in your javascript\r\n$(function() {\r\n $('.input-row #post_topics_list, .input-row #post_tags_list').select2({tags:[]})\r\n});\r\n[/sourcecode]\r\n\r\nAnd you get a beautiful input box with handy tag handling.\r\n\r\n\r\nUpdate:\r\nIf you haven't found it, there's a nice package that puts select2 in the asset pipeline - https://github.com/argerim/select2-rails.

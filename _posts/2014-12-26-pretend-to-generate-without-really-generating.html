\n\nWhen running 'rails generate', you can add a -p (pretend) flag at the end of the command to see what the output will look like, without actually running it. Very handy!\r\n

rails generate model todo_item todo_list:references content:string -p\r\n

\r\n

The console will then show you what would happen if you ran the command for real. In this particular case, my console returned this:

\r\n

invoke active_record\r\ncreate db/migrate/20141206004158_create_todo_items.rb\r\ncreate app/models/todo_item.rb\r\ninvoke rspec\r\ncreate spec/models/todo_item_spec.rb

\r\nThen, if everything looks fine, you can just press the up arrow (to repeat your last command), delete the '-p' and press enter, to run the command for real.
